Design do CronJob:
- **Schedule:** `0 0 * * *` (meia-noite UTC, diário)
- **Namespace:** `cell`
- **Imagem:** `harbor.hasslab.pro/cell/cell-backend:latest`
- **Comando:** `python scripts/extract.py && python scripts/migrate_history.py`
  - extract.py puxa dados da API celula.in → artifacts/extract/
  - migrate_history.py carrega artifacts → Postgres (upserts idempotentes)
- **Envs:** todas do Secret `cell-secret` via `envFrom` (DATABASE_URL + CELULA_*)
- **restartPolicy:** OnFailure
- **backoffLimit:** 2 (retry 2x antes de falhar)
- **successfulJobsHistoryLimit:** 3
- **failedJobsHistoryLimit:** 3
- **concurrencyPolicy:** Forbid (não roda 2 jobs simultâneos)

Pull secret: `harbor-pull-secret` (já existe no namespace para o backend)

Observação sobre artifacts/:
O extract.py grava arquivos em artifacts/extract/ dentro do container.
migrate_history.py lê esses mesmos arquivos. Como ambos rodam no mesmo Pod/container
sequencialmente, o filesystem efêmero do Pod é suficiente — não precisa de PersistentVolume.
Os artifacts são transitórios (recriados a cada run).
</context>

  <action>
Criar `k8s/etl-cronjob.yaml` com o seguinte conteúdo:

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: cell-etl
  namespace: cell
  labels:
    app: cell-etl
spec:
  schedule: "0 0 * * *"
  concurrencyPolicy: Forbid
  successfulJobsHistoryLimit: 3
  failedJobsHistoryLimit: 3
  jobTemplate:
    spec:
      backoffLimit: 2
      template:
        metadata:
          labels:
            app: cell-etl
        spec:
          restartPolicy: OnFailure
          imagePullSecrets:
            - name: harbor-pull-secret
          containers:
            - name: etl
              image: harbor.hasslab.pro/cell/cell-backend:latest
              command:
                - /bin/sh
                - -c
                - |
                  set -e
                  echo "[ETL] Starting extract..."
                  python scripts/extract.py
                  echo "[ETL] Extract complete. Starting load..."
                  python scripts/migrate_history.py
                  echo "[ETL] Pipeline complete."
              envFrom:
                - secretRef:
                    name: cell-secret
              resources:
                requests:
                  memory: "256Mi"
                  cpu: "100m"
                limits:
                  memory: "1Gi"
                  cpu: "500m"
```

Notas:
- `set -e` garante que o job falha se extract.py falhar (não roda migrate com dados parciais)
- `envFrom secretRef cell-secret` carrega DATABASE_URL + todas as vars CELULA_*
- `imagePullSecrets: harbor-pull-secret` — mesmo secret usado pelo backend deployment
- Resources: request conservador, limit com folga para o Playwright base image
